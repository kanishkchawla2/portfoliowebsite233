Technical Screening & Market Structure Engine using Python and Excel

Research
Excel-based market structure dashboard with automated data flow and screening.

A Python-powered Excel engine built to systematically screen 1900+ NSE stocks, map market structure, and identify emerging leadership through a disciplined, repeatable framework.

Overview

With more than 1900 stocks listed on the National Stock Exchange of India, access to data is not the competitive advantage. The advantage lies in structuring that data into a repeatable process that filters noise, improves speed, and highlights leadership early.

This system was built to transform Excel from a static spreadsheet tool into a scalable market analysis engine. By integrating automated Python data pipelines with a structured Excel front-end, the platform enables full-market coverage with minimal manual intervention.

Architecture & Data Stack

  • Python-based backend for automated historical and live OHLCV data fetching.
  • Structured CSV database for clean, append-only historical storage.
  • Power Query pipelines to ingest and transform CSV data directly into Excel.
  • VBA automation for one-click refresh, processing, and structured list generation.
  • Advanced Excel functions including VSTACK, FILTER, INDEX, and CHOOSECOLS for dynamic dataset construction.
  • Live data loop for real-time monitoring and alignment.
  • Excel front-end for visualization, screening logic, and execution tracking.

Every trading day begins with executing master.py, which updates the historical database up to T-1. Once completed, the live engine initiates real-time data flow. Excel then processes the updated datasets with a single refresh action, recalculating indicators and updating screening outputs instantly.

Market Structure Mapping

Each stock is classified by sector, industry, and market capitalization. This enables multi-layered screening and allows observation of capital rotation across different segments of the market.

  • Sector-wise trend identification
  • Industry-level leadership tracking
  • Market-cap participation analysis
  • Broad market breadth observation

Custom Screening Framework

  • Volatility Contraction Pattern (VCP) detection logic built in Python (currently in beta).
  • Trend and consolidation structure filters for early-stage setup identification.
  • Breakout readiness tracking using predefined technical conditions.
  • Multi-stage shortlist generation for structured manual review.

The VCP module is developed in Python and is currently in beta, assisting in narrowing down structurally strong contraction setups. However, the final evaluation remains discretionary. The system supports decision-making, but interpretation and validation are performed manually to maintain contextual judgment.

Execution Framework

The platform integrates into a structured but actively managed decision pipeline. Market bias is assessed first. Stocks move into a Check list based on structural alignment. From there, deeper shortlisting occurs across multiple layers before a stock reaches the Final list.

Fundamental validation, broader market context, and discretionary chart interpretation are performed manually. Only after personal confirmation of structure and trigger alignment does a stock transition into the Buy1 category. The system assists in filtering and organizing data, but majority of the critical decisions and refinements are executed independently.

Impact

The system enables screening of the entire NSE universe in approximately 20–30 minutes while maintaining structure and consistency. It improves speed, reduces noise, and supports data-driven decision making in a repeatable manner.

AI-Enhanced Relative Valuation Scatter Dashboard

Valuation
Standard Relative Valuation Scatter Plot
AI-Enhanced Scatter Plot with Similarity-Based Clustering

A Streamlit-powered interactive dashboard that visualizes relative valuation positioning across peer groups using financial multiples and performance drivers.

Overview

The dashboard fetches financial and market data using yfinance and dynamically plots cross-sectional valuation relationships such as P/E vs Earnings Growth, EV/EBITDA vs ROCE, and other driver-based multiple comparisons.

Instead of static peer comparison, the system enables visual identification of valuation dispersion, relative premiums, and potential mispricing across industry groups.

Core Capabilities

  • Matplotlib-based scatter engine integrated into a Streamlit dashboard.
  • Cross-sectional valuation mapping across industries and custom stock lists.
  • Outlier detection through driver-based multiple comparison.
  • One-click Excel-to-dashboard integration for instant visualization from screened lists.
  • Selected stock is highlighted in red for immediate visual reference against peer clusters, while AI-grouped similar stocks are colored in yellow.

AI-Driven Business Similarity Layer

An advanced AI layer is integrated using Ollama to enhance peer comparison. The model analyzes business descriptions and groups structurally similar companies using semantic similarity scoring.

  • Embedding-based similarity scoring across selected peer sets.
  • Automatic clustering of comparable companies.
  • Color-coded grouping on scatter plots to visualize business proximity.
  • Interactive threshold filtering to refine comparable universes.
  • AI-generated reasoning for similarity relationships.

This transforms traditional multiple comparison into an AI-assisted comparable intelligence system, ensuring that valuation comparisons are made between structurally aligned businesses rather than arbitrary peers.

Impact

The dashboard combines quantitative dispersion analysis, AI-based business clustering, and interactive visualization into a unified research tool. It enables faster identification of valuation inefficiencies while maintaining contextual accuracy in peer comparison.

AI-Powered Concall Summariser

AI Research
AI-assisted concall extraction and summarisation pipeline integrated with Excel and Streamlit.

An automated AI-driven pipeline designed to scrape, optimise, and summarise earnings call transcripts across the NSE universe.

Overview

This system scrapes earnings call transcripts directly from the Screener.in website and processes them through a structured keyword-density engine before generating AI summaries.

Pipeline Architecture

  • Scrapes concall transcripts from Screener.in.
  • Applies high-density keyword extraction logic to capture relevant context windows around critical financial terms.
  • Optimises the AI input by extracting text segments before and after targeted keywords to reduce token load.
  • Feeds structured context into Ollama (currently using a cloud-based model; can be shifted to local models depending on system specifications).
  • Generates structured summaries across 1900+ stocks in approximately one hour.
  • One-click summary viewing directly from the Excel dashboard via an integrated Streamlit window.

Impact

The concall summariser dramatically reduces manual transcript review time while preserving analytical depth. By optimising context windows before AI processing, the system improves efficiency, reduces cost, and enables scalable coverage across the entire market universe.
